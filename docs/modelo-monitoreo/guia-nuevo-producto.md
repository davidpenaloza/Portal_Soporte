# Guía para nuevo producto

## Objetivo

Definir el proceso para incorporar un nuevo producto al modelo de monitoreo y dejarlo operable por soporte.

## Checklist

- Producto registrado en el portal.
- Fuentes asociadas y relaciones producto-fuente documentadas.
- Funciones KQL o wrappers mínimos definidos.
- Dashboard Grafana con criterios OK / WARN / ALERT.
- Runbook y matriz de escalamiento asociados.
- Evidencia mínima de traspaso a soporte.

## Cierre

El producto debe quedar con responsable, ambiente soportado, frecuencia esperada, dashboard principal y documentación disponible en el visor del portal.
